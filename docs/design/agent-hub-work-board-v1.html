Agent Hub Plugin UI — Detached Window Scenario Storyboard

이 정적 목업은 공개 웹사이트나 웹 대시보드가 아니라, LVIS 앱에서 Agent Hub 플러그인을 열었을 때 분리되는 Electron/plugin detached window의 상태 변화를 보여준다. 로그인 구현, callback 안정화, 실제 데이터 연동을 완료했다고 주장하지 않으며, 사용 여정별 화면 합의를 위한 디자인 산출물이다.

Plugin chrome assumptions
플러그인 창은 LVIS 메인 앱에서 분리된 compact viewport로 열린다.
실제 웹 로그인은 브라우저/외부 창에서 진행되고, 플러그인 UI는 대기 상태를 보여준다.
아래의 callback/sync 화면은 목표 UX state이며 구현 완료 표시가 아니다.

Recovery copy principle

사용자는 “무엇이 실패했는지”, “현재 어떤 데이터가 보이는지”, “다음으로 무엇을 누르면 되는지”를 플러그인 창 안에서 바로 알아야 합니다.

Design discussion notes · 사용자 합의 필요

  • Detached plugin window의 layout density: 한 화면에 많이 보여줄지, 카드 단위 scrolling을 허용할지.
  • 초기 진입 후 기본 탭을 My Work first로 할지, Team Board first로 할지.
  • 웹 로그인 waiting copy: “브라우저에서 완료” 중심인지, “플러그인으로 돌아오기” 중심인지.
  • board card grouping: source별(Agent Hub/MS Graph/Meeting)인지, 업무 맥락별(오늘/주간/팀)인지.
  • team board shape: 팀장님 한마디를 항상 상단 고정할지, 리스크가 높을 때만 강조할지.

Scope notes · 구현으로 오해하지 말 것

  • 이 파일은 정적 docs/design artifact이며 runtime/source/plugin 파일을 변경하지 않는다.
  • callback 수신과 source sync는 UX 합의를 위한 state 표현이며 auth callback 구현 완료를 의미하지 않는다.
  • 실제 로그인 페이지/브라우저는 side callout으로만 표현하고, 주 디자인 대상은 Agent Hub Plugin UI이다.
  • 플러그인 버전 bump는 필요 없다. docs/design 전용 변경이다.
