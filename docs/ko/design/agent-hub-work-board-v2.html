Agent Hub Plugin UI v2 — 알약 토글 기반 detached window

v1 합의 사항을 반영한 두 번째 목업입니다. 사이드바를 제거하고 상단의 마이워크 / 팀보드 알약 토글과 우측 상단 설정 톱니로 IA 를 단순화했습니다. 인증 라이프사이클(S0~S2)은 한 화면 안에서 상태가 전이되는 모습으로 통합했고, S3 에서 회의록 항목은 제거했습니다 — 회의록은 Meeting plugin 의 책임 영역이며 Agent Hub 보드의 1차 표면이 아닙니다.

Recovery copy principle

사용자는 “무엇이 실패했는지”, “현재 어떤 데이터가 보이는지”, “다음으로 무엇을 누르면 되는지”를 플러그인 창 안에서 바로 알아야 합니다.

v2 합의 필요 항목

  • 알약 토글의 키보드 패턴: Tab → Arrow Left/Right (segmented control 표준) vs Tab → Tab (개별 button).
  • 설정 톱니 진입 시: 모달인지, 우측 sliding panel 인지, 같은 plugin window 안에서의 view 전환인지.
  • 인증 단계에서 토글이 비활성일 때 사용자가 클릭하면: silent ignore 인지, “먼저 로그인이 필요합니다” tooltip 인지.
  • 마이워크 ↔ 팀보드 전환 시 scroll 위치: 각 view 별 독립 보존 vs 공통 reset.
  • S2 sync 완료 직후 기본 view: 마이워크 first (일관) vs 마지막 사용 view 복원.

Scope notes · 구현으로 오해하지 말 것

  • 이 파일은 정적 docs/design artifact 이며 runtime/source/plugin 파일을 변경하지 않는다.
  • 회의록 제거는 Agent Hub 보드 표면에서의 제거를 의미하며, Meeting plugin 자체 UI/책임은 그대로 유지된다.
  • 알약 토글 / 톱니 / 사이드바 제거는 IA 합의 산출물이며, 컴포넌트 구현은 후속 PR.
  • 플러그인 버전 bump 는 필요 없다. docs/design 전용 변경이다.
